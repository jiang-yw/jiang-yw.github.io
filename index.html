评分预测算法

平均值

最简单的评分预测算法是利用平均值预测用户对物品的评分的。

基于邻域的算法

  • 基于用户：预测一个用户对一个物品的评分，参考和这个用户兴趣相似的用户对该物品的评分
  • 基于物品：预测一个用户对一个物品的评分，参考用户对这个物品相似的其他物品的评分

隐语义模型和矩阵分解模型

二者都是通过降维的方法将评分矩阵补全。

用户的评分行为可以表示成一个评分矩阵 $R$，其中 $R[u][i]$ 是用户 $u$ 对物品 $i$ 的评分。用户不会对所有物品评分，所以矩阵很多元素都是空的，空的元素成为 缺失值（missing value）。如果一个用户对物品没有评分，推荐系统需要预测这个用户是否会对这个物品评分以及评几分。

  • 推荐系统联系用户和物品的途径

特征表现形式

  1. 物品的属性集合：例如图书的属性集合包括作者、出版社、主题和关键词等
  2. 隐语义模型
  3. 标签：描述物品的语义

标签分类

  1. 作者或专家给用户打标签
  2. 用户给物品打标签，即 UGC（User Generated Content，用户生成的内容）
    用户对一个物品打上一个标签，这个标签既描述了用户的兴趣，又表示了物品的语义，从而将用户和物品联系起来。

标签系统中的推荐问题

  1. 如何利用用户打标签的行为为其推荐物品（基于标签的推荐）？
  2. 如何在用户给物品打标签时为其推荐适合该物品的标签（标签推荐）？

基于标签的推荐系统

算法优化

  1. 标签扩展
    对每个标签找到和它相似的标签，计算标签之间的相似度。
  2. 标签清理
    1. 去除词频很高的停止词（或表示情绪的词）
    2. 去除因词根不同造成的同义词
    3. 去除因分隔符造成的同义词

《推荐系统实践》读书笔记（3）

发表于 2018-04-04

冷启动问题

  1. 用户冷启动：给新用户做个性化推荐。
  2. 物品冷启动：将新的物品推荐给它感兴趣的用户。
  3. 系统冷启动：在一个新开发的网站上（还没有用户，也没有用户行为，只有一些物品的信息）设计个性化推荐系统。

冷启动解决方案

  1. 提供非个性化的推荐：热门排行榜
  2. 利用用户注册信息做粗粒度的个性化
  3. 利用用户社交账号登录，导入用户社交网站的好友信息，给用户推荐其好友喜欢的物品。
  4. 要求用户登录时对一些物品进行反馈，收集用户对这些物品的兴趣信息，推荐相似物品。
  5. 对于新加入的物品，利用内容信息，将其推荐给喜欢过其相似物品的用户。
  6. 系统冷启动时，引入专家知识，建立物品的相关度表。

《推荐系统实践》读书笔记（2）

发表于 2018-04-03

用户行为数据

  • 原始日志（raw log）按照用户行为汇总成会话日志（session log），会话日志通常存储在分布式数据仓库中，例如 Hadoop Hive（离线分析） 和 Google Dremel（在线分析）。
  • 用户行为一般分为两种：
    1. 显性反馈行为（explicit feedback）：主要方式为评分或喜欢/不喜欢。
    2. 隐性反馈行为（implicit feedback）：最具代表性：页面浏览行为。
  • 显隐性反馈数据比较：
— 显性反馈数据 隐性反馈数据
用户兴趣 明确 不明确
数量 较少 庞大
存储 数据库 分布式文件系统
实时读取 实时 有延迟
正负反馈 都有 只有正反馈
  • 用户行为的统一表示
user id 产生行为的用户唯一标识
item id 产生行为的对象唯一标识
behavior type 行为的种类
context 产生行为的上下文，时间和地点等
behavior weight 行为的权重
behavior content 行为的内容

用户行为分析

  • 长尾分布：用户越活跃，越倾向于浏览冷门的商品。
  • 协同过滤算法：基于邻域的方法（neighborhood-based）、隐语义模型（latent factor model）、基于图的随机游走算法（random walk on graph）

协同过滤算法

数据集

利用 MovieLens 数据集（忽略评分记录）研究隐反馈数据集 TopN 推荐问题，即预测用户会不会对某部电影评分，而非预测用户对某电影评多少分。

离线实验设计
  1. 将用户行为数据集按均匀分布随机分成M份，选一份为测试集，其余为训练集。
  2. 在训练集上建立用户兴趣模型，并在测试集上对用户行为进行预测，统计出相应的评价指标。
  3. 为保证指标不过拟合，需进行 M 次实验，并每次选用不同测试集，将 M 次实验测出的评测指标的平均值作为最终评测指标。
评测指标
  1. 召回率：描述有多少比例的用户-物品评分记录包含在最终的推荐列表中。
  2. 准确率：描述最终的推荐列表中有多少比例是发生在用户-物品评分记录。
  3. 覆盖率：反映推荐算法发掘长为的能力，覆盖率越高，越能将长尾中的物品推荐给用户。
  4. 新颖度：对每个物品的流行度取对数，计算平均流行度。（因物品流行度分布满足长尾分布，取对数使得流行度平均值更稳定。）
基于邻域的算法
  1. 基于用户的协同过滤算法（UserCF）：给用户推荐和他兴趣相似的其他用户喜欢的物品。
  2. 基于物品的协同过滤算法（ItemCF）：给用户推荐和他之前喜欢的物品相似的物品。
隐语义模型

对物品和用户兴趣进行分类。先得到用户的兴趣分类，再从分类中推荐用户可能喜欢的物品。

基于图的模型

将用户行为用图（二分图）表示，并利用图的算法给用户进行个性化推荐。

发表于 2018-04-02

学术论文把推荐问题看做评分预测问题，而实际应用中最常见的是 TopN 推荐。

推荐算法

  1. 社会化推荐（social recommendation）
  2. 基于内容推荐（content-based filtering）
  3. 基于协同过滤的推荐（collaborative filtering）：历史兴趣相似的用户

推荐系统实验方法

  • 离线实验（offline experiment）
    • 步骤
      (1)通过日志系统获得用户行为数据，并按照一定格式生成一个标准的数据集
      (2)将数据集按照一定的规则分成训练集和测试集
      (3)在训练集上训练用户兴趣模型，在测试集上进行预测
      (4)通过事先定义的离线指标评测算法在测试集上预测结果
    • 优缺点
优点 缺点
不需要用户有对实际系统的控制权 无法计算商业上关心的指标
不需要用户参与实验 离线实验的指标和商业指标存在差距
速度快，可以测试大量算法 ———
  • 用户调查（user study）
  • 在线实验（online experiment）
    • AB 测验：通过一定的规则将用户随机分成几组，并对不同组的用户采用不同的算法，通过统计不同组用户的不同评测指标来比较不同算法

评价指标

  1. 用户满意度
    无法离线计算，只能通过用户调查或在线实验获得。
  2. 预测准确度
    可通过离线计算。
    2.1 评分预测
    通过均方根误差（RMSE）和平均绝对误差（MAE） 计算。
    2.2 TopN 推荐
    通过准确率（precision）/召回率（recall）度量。
    有时，为全面评测 TopN 推荐的准确率和召回率，选取不同的推荐列表长度 N，计算出一组准确率/召回率，然后画出准确率/召回率曲线（precision/recall curve）。
  3. 覆盖率（coverage）
    最简单定义为推荐系统能够推荐出来的物品占总物品集合的比例。
    两个著名指标：信息熵和基尼系数。
  4. 多样性
  5. 新颖性
  6. 惊喜度（serendipity）
  7. 信任度（trust）
  8. 实时性
  9. 健壮性
  10. 商业目标

评价维度

  1. 用户维度
  2. 物品维度
  3. 时间维度
